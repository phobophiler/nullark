## Runtime record

Public runtime fields can be compared against `public-artifacts/current.json`. Approval booleans are scoped by `approvalEvidence` and `approvalSemantics`; they are not standalone production-readiness claims.

## Operator-action boundary

The runtime record approves the exact pinned v1.2 public runtime binding. It is not approval for:

- moving mainnet funds outside the pinned v1.2 runtime binding
- signing transactions from this checkout
- funding relayers or operators
- rolling out guarded-user cohorts without private operator evidence
- enabling or changing production relayers beyond the pinned endpoint and relayer binding
- changing fee policy
- presenting production privacy as enabled
- treating a checkout as production-ready

<Warning>
  If private operator evidence is missing, stale, or unavailable, production readiness remains unproven.
</Warning>
