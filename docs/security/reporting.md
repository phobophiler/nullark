# Reporting Security Issues

Do not open a public issue for vulnerabilities that could move funds, expose note material, bypass relayer checks, replay signatures, or weaken proof verification.

Send a private report to the maintainer with:

- affected component and file path
- network and chain ID
- exploit preconditions
- impact
- minimal reproduction
- whether private keys, note material, raw witnesses, proof blobs, calldata, or live transaction data are involved

Do not include seed phrases, private keys, raw note secrets, wallet unlock signatures, or unredacted user data.
